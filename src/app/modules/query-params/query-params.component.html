QueryParams

  • We can send some query params in our routes.

  • Eg: -

    /product?search=20&limit=10

  • We can then read these query params in the component class and process them by subscribing the queryParams.

  • Mostly used for searching, filtering, querying data etc.

  • Eg: -

    This "id: {{ id }}" is coming from the query param.

  • We can access multiple query params from the URL.
