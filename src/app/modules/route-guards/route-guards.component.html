RouteGuards works!

RouteGuards

  • They help us to prevent the user to access the routes without authentication.

  • Used to secure route paths.

  • Route guards resolve to true or false based on a custom logic/functionality.

  • We can write any number of route guards.

  • Syntax: - ng g guard guard_name

  • Guards have the following interfaces which we can use: -

    • canActivate - tells if we can use the parent route or not.

    • canActivateChild - tells if we can use the child route or not.

    • canDeactivate - tells if we can leave the route or not.

    • canLoad - can the lazy loaded module be loaded or not.

    • resolve - tells what to do before the routes get activated.
